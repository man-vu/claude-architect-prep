Saber lo que **no** entra en el examen te evita estudiar de más. Estos temas adyacentes están explícitamente fuera del alcance de Claude Certified Architect — Foundations.

## Lo que no entra en el examen
- Ajuste fino (fine-tuning) de modelos de Claude o entrenamiento de modelos personalizados
- Autenticación, facturación o gestión de cuentas de la Claude API
- Implementación detallada en lenguajes/frameworks específicos (más allá de lo necesario para la configuración de herramientas/esquemas)
- Despliegue u hospedaje de servidores MCP (infraestructura, redes, orquestación de contenedores)
- La arquitectura interna de Claude, el proceso de entrenamiento o los pesos del modelo
- Constitutional AI, RLHF o metodologías de entrenamiento de seguridad
- Modelos de embeddings o detalles de implementación de bases de datos vectoriales
- Computer use (automatización de navegador, interacción con el escritorio)
- Análisis de imágenes / capacidades de visión (Vision)
- Streaming API o server-sent events
- Límites de tasa (rate limiting), cuotas o cálculos detallados de costos de la API
- OAuth, rotación de claves de API o detalles de protocolos de autenticación
- Configuraciones específicas de proveedores de nube (AWS, GCP, Azure)
- Benchmarks de rendimiento o métricas de comparación de modelos
- Detalles de implementación del prompt caching (más allá de saber que existe)
- Algoritmos de conteo de tokens o detalles de tokenización

## Por qué esto importa
El examen evalúa el criterio para decidir trade-offs arquitectónicos con las cuatro tecnologías fundamentales — no la infraestructura, los detalles internos del modelo ni las operaciones específicas de cada proveedor. Si una pregunta parece depender de alguno de los temas anteriores, vuelve a leerla: la respuesta esperada casi siempre trata sobre los componentes fundamentales de Claude.
